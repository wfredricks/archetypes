# graph-db — Reference Implementation (pre-lift pointer)

**Canonical reference: [`wfredricks/polygraph`](https://github.com/wfredricks/polygraph)**

This archetype is a **stub** in this registry — see the directory's `ARCHETYPE.md`
for the trigger condition that will produce the full lift. The polygraph library
already exists; multiple constellation Neo4j volumes already store
polygraph-shaped data (see `udt-v01_constellation-neo4j` etc.) but no service has
yet adopted polygraph under the recipe-file build discipline that produces a
registry-ready archetype lift.

When that adoption happens, this pointer will be joined by a full
`ARCHETYPE.md`, `ADOPTION-RECIPE.md`, `DEFECTS.md`, and `ADOPTIONS.md`.

Until then, treat the polygraph repo itself as the source of truth for the
pattern.
